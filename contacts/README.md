## Schema (8 columns)

```
email,name,designation,category,tags,case,source,mobile
```

- **email** — unique key, lowercase.
- **tags** — `|`-separated, lowercase. Used for sorting/segmenting campaigns.
- **case** — `general` or a client slug (`tarun-thadani`, `olympio-almeida`, …).
- **mobile** — public office/helpline/WhatsApp number only (personal mobiles not sourced — DPDP).

## Maintenance scripts

| Script | What it does |
|---|---|
| `clean_and_audit.py` | **Canonical tool.** Normalises tags/fields, dedupes (unions tags), quarantines bad syntax, live MX/A-verifies every domain (public DNS 8.8.8.8/1.1.1.1 + retry), rebuilds live, prints full summary. Run after any bulk change. |
| `verify_all.py` | Inbox bounce sweep (4 ProtonMail accounts) + syntax + DNS. Run to catch new bounces. |
| `dedupe_contacts.py` | Collapse duplicate emails only. |
| `recheck_dead_domains.py` | Re-validate `dead_domain` suppressions, recover false positives. |
| `expand_*.py` | Append batches (Mumbai prominent, crime network, Maharashtra police/courts/admin). All MX-verify before adding; never add guessed local-parts. |

## Rules (deliverability — non-negotiable)

1. **Never add guessed email local-parts.** A 43% bounce suspended ZeptoMail + Resend. Only add addresses copied verbatim from official directories or that pass live verification.
2. **Always send from `contacts_live.csv`** (suppression already applied); re-check `suppression_list.csv` at send time.
3. **Personal mobile numbers are not sourced** — only published office/helpline/WhatsApp numbers (DPDP Act compliance).
4. Run `clean_and_audit.py` before any major campaign and after any bulk import.
